Claude Board'daki her gorev, Jira sorun anahtarlarina benzer sekilde `{TUR}-{PROJE}-{NUMARA}` formatinda otomatik olusturulan bir anahtar alir.

<Frame><img src="/images/feature-task-keys.svg" alt="Task Keys" /></Frame>

## Format

```
FTR-CB-1001
 │   │   │
 │   │   └── Proje basina sirali sayac (1001'den baslar)
 │   └────── Proje anahtari (proje slug'indan turetilir)
 └────────── Gorev turu on eki
```

## Tur On Ekleri

| Gorev Turu | On Ek |
|------------|-------|
| Feature | `FTR` |
| Bug Fix | `BUG` |
| Refactor | `RFT` |
| Docs | `DOC` |
| Test | `TST` |
| Chore | `CHR` |

## Proje Anahtari

Proje anahtari, proje slug'indan otomatik olarak turetilir:

- Cok sozcuklu slug'lar bas harfleri kullanir: `claude-board` → `CB`, `utd-games-website` → `UGW`
- Tek sozcuklu slug'lar ilk 3 karakteri kullanir: `renkler` → `RNK`

## Anahtarlar Nerede Gorunur

Gorev anahtarlari, daha once gorev kimliginin gosterildigi her yerde goruntulenir:

- Kanban board'undaki **gorev kartlari**
- **Liste gorunumu** satirlari
- **Zaman cizgisi gorunumu** etiketleri
- **Gorev detay penceresi** basligi
- **Canli terminal** basligi

## Otomatik Guncellemeler

- Anahtarlar gorev olusturulurken uretilir
- Bir gorevin turu degistiginde (orn. feature → bugfix), on ek otomatik olarak guncellenir (`FTR-CB-1001` → `BUG-CB-1001`)
- Mevcut gorevler sunucu baslatildiginda anahtarlarla geriye donuk olarak doldurulur
