Claude Board'daki her görev, Jira sorun anahtarlarına benzer şekilde `{TÜR}-{PROJE}-{NUMARA}` formatında otomatik oluşturulan bir anahtar alır.

<Frame><img src="/images/feature-task-keys.svg" alt="Task Keys" /></Frame>

## Format

```
FTR-CB-1001
 │   │   │
 │   │   └── Proje başına sıralı sayaç (1001'den başlar)
 │   └────── Proje anahtarı (proje slug'ından türetilir)
 └────────── Görev türü ön eki
```

## Tür Ön Ekleri

| Görev Türü | Ön Ek |
|------------|-------|
| Feature | `FTR` |
| Bug Fix | `BUG` |
| Refactor | `RFT` |
| Docs | `DOC` |
| Test | `TST` |
| Chore | `CHR` |

## Proje Anahtarı

Proje anahtarı, proje slug'ından otomatik olarak türetilir:

- Çok sözcüklü slug'lar baş harfleri kullanır: `claude-board` → `CB`, `utd-games-website` → `UGW`
- Tek sözcüklü slug'lar ilk 3 karakteri kullanır: `renkler` → `RNK`

## Anahtarlar Nerede Görünür

Görev anahtarları, daha önce görev kimliğinin gösterildiği her yerde görüntülenir:

- Kanban board'undaki **görev kartları**
- **Liste görünümü** satırları
- **Zaman çizgisi görünümü** etiketleri
- **Görev detay penceresi** başlığı
- **Canlı terminal** başlığı

## Otomatik Güncellemeler

- Anahtarlar görev oluşturulurken üretilir
- Bir görevin türü değiştiğinde (örn. feature → bugfix), ön ek otomatik olarak güncellenir (`FTR-CB-1001` → `BUG-CB-1001`)
- Mevcut görevler sunucu başlatıldığında anahtarlarla geriye dönük olarak doldurulur
