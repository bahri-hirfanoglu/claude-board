Bir görev **In Progress** durumuna geçtiğinde, Claude Board yerel makinenizde bir Claude Code CLI süreci başlatır. Bu "ajan"dır — kod tabanınızı okur, dosyalara yazar, komutlar çalıştırır ve kod commit'ler.

## Ajanların Başlatılması

<Steps>
  <Step title="Görev In Progress durumuna geçer" icon="play">
    Sunucu, görev açıklamasından, proje [bağlam parçalarından](/features/snippets), [prompt şablonundan](/features/prompt-templates), [rolden](/features/roles) ve varsa [dosya eklerinden](/features/attachments) bir prompt oluşturur.
  </Step>
  <Step title="Dal oluşturulur" icon="code-branch">
    Git otomasyonu etkinse, temel daldan bir özellik dalı oluşturulur (örneğin `feature/CB-101-add-auth`).
  </Step>
  <Step title="CLI süreci başlatılır" icon="terminal">
    Claude Board, `claude` komutunu `--no-input` ve yapılandırılmış izin moduyla çalıştırır. Çalışma dizini, projenin yapılandırılmış yoluna ayarlanır.
  </Step>
  <Step title="Akış başlar" icon="signal">
    Tüm çıktı yakalanır ve WebSocket aracılığıyla bağlı istemcilere aktarılır. Günlükler, [oturum tekrarı](/features/session-replay) için veritabanına kaydedilir.
  </Step>
  <Step title="Ajan tamamlanır" icon="check">
    CLI çıkış yaptığında, görev otomatik olarak **Testing** durumuna geçer. [Otomatik test](/features/auto-test) etkinse, geçişten önce testler çalıştırılır.
  </Step>
</Steps>

## Token Takibi

Claude Board, görev başına token kullanımını gerçek zamanlı olarak takip eder:

| Metrik | Açıklama |
|--------|-------------|
| **Girdi token'ları** | Claude'a gönderilen token'lar (prompt + bağlam) |
| **Çıktı token'ları** | Claude tarafından üretilen token'lar |
| **Maliyet tahmini** | Model fiyatlandırmasından hesaplanır (Opus/Sonnet/Haiku oranları) |
| **Önbellek okuma/oluşturma** | Prompt önbelleğinden sunulan veya eklenen token'lar |

Kullanım güncellemeleri `task:usage` WebSocket olayıyla yayılır ve görev kartında görünür.

## Akıllı Zamanlayıcı

Görev zamanlayıcısı, aktif ajan süresini doğru bir şekilde ölçer:

| Olay | Zamanlayıcı |
|-------|-------|
| Görev In Progress durumuna geçer | **Başlar** |
| Görev Testing durumuna geçer (inceleme bekliyor) | **Duraklar** |
| Görev In Progress durumuna döner (değişiklik istendi) | **Devam eder** |
| Görev Done durumuna geçer | **Durur** |

<Info>Akıllı zamanlayıcı, inceleme bekleme süresini hariç tutarak Claude'un gerçekte kodlamaya ne kadar zaman harcadığının doğru bir resmini sunar.</Info>

## Eşzamanlılık

Aynı anda en fazla **5 ajan** çalıştırabilirsiniz. [Otomatik kuyruk](/features/queue) özelliği, proje başına eşzamanlılık sınırı belirlemenize olanak tanır.

| Ajan Sayısı | En Uygun Kullanım |
|--------|----------|
| **1** | Sıralı görevler, basit projeler |
| **2-3** | Düşük çatışma riskiyle orta düzeyde paralellik |
| **4-5** | [Bağımlılık yönetimi](/features/dependencies) ile maksimum verimlilik |

<Warning>Aynı projede birden fazla ajan çalıştırmak dosya çatışmalarına neden olabilir. Her ajanın hangi dosyalara eriştiği koordine etmek için [görev bağımlılıklarını](/features/dependencies) kullanın. [Canlı görünüm](/features/orchestration) çatışmaları gerçek zamanlı olarak tespit eder.</Warning>

## Ajan Yaşam Döngüsü Olayları

Her ajan, izleyebileceğiniz olaylar yayar:

| Olay | Ne Zaman |
|-------|------|
| `task:started` | Ajan süreci başlatıldığında |
| `task:output` | Terminal çıktı parçası |
| `task:usage` | Token/maliyet güncellemesi |
| `task:tool_call` | Bireysel araç çağrısı (Read, Write, Edit, Bash) |
| `task:completed` | Ajan başarıyla çıkış yaptığında |
| `task:failed` | Ajan hatayla çıkış yaptığında |
| `agent:file_conflict` | İki ajan aynı dosyaya eriştiğinde |

WebSocket abonelik ayrıntıları için [Events API](/api/events) sayfasına bakın.
