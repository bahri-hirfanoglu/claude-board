Planning Mode uses Claude to analyze a topic and automatically break it down into structured, actionable tasks on your board. Instead of manually creating tasks, describe what you want to build and let Claude explore your codebase, research the approach, and generate a complete task breakdown.

<Frame><img src="/images/feature-planning.svg" alt="Planning Mode Workflow" /></Frame>

## How It Works

1. Open Planning Mode from the project header
2. Describe what you want to build
3. Configure model, effort, and granularity
4. Claude explores your codebase, then generates tasks
5. Tasks are created in your Backlog automatically

## Configuration Options

### Task Granularity

Control how Claude breaks down the work:

| Level | Tasks | Style |
|-------|-------|-------|
| **High-level** | 3-5 | Major milestones with sub-step checklists |
| **Balanced** | 5-10 | Meaningful units grouping related changes |
| **Detailed** | 10-20 | Atomic tasks, each completable in a single session |

### Model Selection

Choose which Claude model powers the planning:

| Model | Best For |
|-------|----------|
| **Haiku** | Quick breakdowns of simple features |
| **Sonnet** | Balanced speed and quality (default) |
| **Opus** | Complex architecture and deep analysis |

### Thinking Effort

Controls Claude's thinking budget:

| Level | Behavior |
|-------|----------|
| **Low** | Fast, surface-level analysis |
| **Medium** | Balanced depth (default) |
| **High** | Deep research and thorough exploration |

## Review & Approve Workflow

Generated tasks are no longer added to your board immediately. Instead, Planning Mode presents them as **proposals** that you can review before committing.

<Steps>
  <Step title="Claude generates proposals" icon="list-check">
    After analyzing your codebase, Claude produces a list of proposed tasks with titles, descriptions, types, and priorities.
  </Step>
  <Step title="Review the breakdown" icon="magnifying-glass">
    Each proposed task is displayed in an expandable card. Read the description, check acceptance criteria, and verify the dependency order.
  </Step>
  <Step title="Approve or revise" icon="check">
    Approve the entire plan to create all tasks at once, or select individual tasks to approve. You can also remove tasks you don't need before approving.
  </Step>
</Steps>

### Revise Capability

If the proposed breakdown doesn't match your expectations, click **Revise** to provide feedback. Describe what should change — different scope, more or fewer tasks, different priorities — and Claude regenerates the plan using your guidance. You can revise as many times as needed before approving.

<Tip>Use the revise step to steer Claude toward your preferred task structure. For example: "Split the API work into separate tasks per endpoint" or "Combine the CSS tasks into one."</Tip>

## Live Progress

While Claude works, the modal shows real-time feedback through an improved activity feed:

- **Phase stepper** — Starting, Exploring, Planning, Reviewing, Done
- **Live stats** — Elapsed time, token usage, tool calls, turns
- **Activity feed** — A structured, scrollable log showing file reads, searches, and commands as they happen, with clear labels and timestamps
- **Text output** — Claude's reasoning as it analyzes the codebase
- **Progress indicators** — Visual progress bars for each phase transition, so you know how far along the planning process is

## Session Persistence

Planning sessions survive page refreshes. If you reload the page while planning is active, the modal automatically reopens and reconnects to the running session. The backend continues processing regardless of the frontend state.

## Generated Tasks

Each generated task includes:

- **Title** — Short, actionable description
- **Description** — Detailed implementation notes with sub-step checklists
- **Type** — `feature`, `bugfix`, `refactor`, `docs`, `test`, or `chore`
- **Priority** — None, Low, Medium, or High
- **Acceptance Criteria** — Definition of done for the task

Tasks are ordered by dependency — prerequisite tasks appear first. Descriptions reference actual files and patterns from your codebase when possible.

<Tip>Add context in the optional field to guide Claude's planning. For example: "Express.js backend, React frontend, PostgreSQL with Prisma ORM" helps Claude generate more relevant tasks.</Tip>
