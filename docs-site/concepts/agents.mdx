When a task moves to **In Progress**, Claude Board spawns a Claude Code CLI process on your local machine. This is the "agent" — it reads your codebase, writes files, runs commands, and commits code.

## How Agents Spawn

<Steps>
  <Step title="Task enters In Progress">
    The server builds a prompt from the task description, project context snippets, prompt template, and any file attachments.
  </Step>
  <Step title="CLI process starts">
    Claude Board runs `claude` with `--no-input` and the configured permission mode. The working directory is set to the project's configured path.
  </Step>
  <Step title="Streaming begins">
    All output is captured and streamed to connected clients via WebSocket. Logs are persisted to the database.
  </Step>
  <Step title="Agent completes">
    When the CLI exits, the task moves to **Testing** automatically.
  </Step>
</Steps>

## Live Terminal

Each running agent has a live terminal you can open from the task card. It shows every action Claude takes — file reads, edits, bash commands, and reasoning. See [Terminal](/features/terminal) for details.

## Token Tracking

Claude Board tracks token usage per task in real-time:

| Metric | Description |
|--------|-------------|
| **Input tokens** | Tokens sent to Claude (prompt + context) |
| **Output tokens** | Tokens generated by Claude |
| **Cost estimate** | Calculated from model pricing |
| **Cache read/creation** | Tokens served from or added to prompt cache |

Usage updates are emitted via the `task:usage` WebSocket event and visible on the task card.

## Smart Timer

The task timer measures active agent time:

- **Starts** when the task enters In Progress
- **Pauses** when the task moves to Testing (awaiting review)
- **Resumes** if the task returns to In Progress after requesting changes
- **Stops** when the task moves to Done

<Info>The smart timer gives you an accurate picture of how long Claude actually spent coding, excluding review wait time.</Info>

## Concurrency

You can run multiple agents simultaneously. The [auto-queue](/features/queue) feature lets you set a maximum concurrency of 1 to 5 agents. Each agent runs in its own process with its own working directory context.

<Warning>Running multiple agents on the same project can cause file conflicts. Use separate branches or coordinate tasks carefully.</Warning>
